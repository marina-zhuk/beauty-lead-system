# Integrations

## Порядок обработки заявки

1. API route принимает данные формы.
2. Валидирует данные через Zod.
3. Отправляет Telegram-уведомление.
4. Возвращает пользователю `success`.

## Telegram

Telegram-уведомление подключено в текущей версии.

Поведение:

1. API route получает валидную заявку.
2. Формирует короткое сообщение для владельца.
3. Отправляет сообщение через Telegram Bot API.
4. Если Telegram API возвращает ошибку, пользователь видит безопасное сообщение без технических деталей.
5. В лог попадает только безопасная информация: статус, описание ошибки или факт отсутствия конфигурации. Токен не логируется.

Переменные окружения:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

## Google Apps Script

Переменная `GOOGLE_APPS_SCRIPT_WEBHOOK_URL` оставлена в `.env.local.example` для будущего сценария без Google Cloud.

В текущей версии она не используется. Если переменная пустая, проект все равно работает через Telegram.

## Google Cloud

Google Cloud и Google Sheets API в текущем MVP не используются.
